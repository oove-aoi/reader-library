# 個人書籍管理系統
## 一、專案簡介

本專案為一套書籍資料管理系統，針對多平台購書與閱讀情境下，書籍資訊分散、系列追蹤困難與閱讀紀錄管理不便等問題進行整合設計。
系統以後端服務為核心，實作完整的資料模型設計與 CRUD 功能，並提供複合查詢與關聯管理能力。

## 二、問題背景

隨著實體書與電子書平台並行發展，使用者可能在不同平台購買相同書籍，或難以有效追蹤系列出版進度。此外，閱讀心得與個人評價往往分散於不同平台或社群媒體，缺乏結構化管理。

### 本系統旨在提供：

書籍與作者資料集中管理

系列出版追蹤

結構化閱讀心得保存

標籤化分類與多條件查詢

追蹤清單建立機制

## 三、核心模組設計

### 系統主要包含以下核心實體與模組：

書籍（Book）

作者（Author）

系列（Series）

標籤（Tag）

閱讀心得（Experience）

追蹤清單（Tracking List）

篩選器生成模組

### 透過關聯設計支援：

作者與書籍一對多關聯

系列與書籍關聯

書籍與標籤多對多關聯

書籍與心得一對多關聯

## 四、已實作功能
### 1. 書籍管理

書籍基本 CRUD

複合條件查詢（依作者、標籤、系列等條件篩選）

### 2. 作者管理

作者基本 CRUD

查詢該作者名下書籍列表

### 3. 標籤管理

標籤基本 CRUD

查詢具特定標籤之書籍清單

### 4. 閱讀心得管理

心得基本 CRUD

與書籍關聯儲存

### 5. 追蹤清單功能

可建立書籍追蹤清單

支援後續擴充出版狀態管理

## 五、技術實作重點:

採用 Spring Boot 建立 RESTful API 架構

使用 Spring Data JPA 進行 ORM 映射與關聯管理

實作雙向與單向關聯設計

採用MVC分層架構（Controller / Service / Repository）

DTO 與 Entity 分離設計

使用Swagger 提供 API 文件
